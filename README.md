# Twitch Ad Blocker（全チャンネル対応）

Twitch の広告（配信に埋め込まれる動画広告・表示広告・トラッキング）をブロックする Chrome 拡張機能（Manifest V3）です。特定チャンネルに限定せず、`twitch.tv` の全チャンネルで動作します。

## なぜ Twitch の広告ブロックは難しいのか

一般的な広告ブロッカー（uBlock Origin など）は「広告用の URL へのリクエストをブロックする」という方式です。バナー広告やポップアップにはこれで十分ですが、**Twitch のライブ配信広告には効きません**。理由は Twitch が **SSAI（Server-Side Ad Insertion / サーバーサイド広告挿入）** を採用しているためです。

- 広告の映像が、**配信本編と同じ動画ストリーム（HLS）の中に直接つなぎ込まれて**配信される
- 広告と本編が**同じドメイン・同じ接続**で届くため、「広告だけをブロック」ができない
- 広告部分だけをブロックすると、映像そのものが止まってしまう

そのため本拡張機能は、単純なブロックではなく **「広告入りストリームを、広告なしストリームに差し替える」** という手法を取ります。

## 全体アーキテクチャ

Twitch の広告は大きく3種類あり、それぞれ別の手法で対処します。

| 広告の種類 | 具体例 | 対処するファイル | 手法 |
|---|---|---|---|
| **動画広告 (SSAI)** | 配信開始時／視聴中に挟まる映像広告（プリロール・ミッドロール） | `worker-hook.js` | ストリーム差し替え |
| **表示広告・トラッキング** | Amazon 広告、解析ビーコン、外部広告SDK | `rules.json` | ネットワークブロック |
| **オーバーレイ／バナー** | 「広告表示中」ラベル、ページ内の表示広告枠 | `content.js` | DOM 要素の除去 |

これらは Chrome 拡張の異なる実行コンテキストで動作します。

```
┌─────────────────────────────────────────────────────────┐
│                  Chrome ブラウザ                          │
│                                                          │
│  ┌────────────────────────────────────────────────┐     │
│  │  declarativeNetRequest (rules.json)             │     │
│  │  → 広告ドメインへの通信をブラウザ層でブロック      │     │
│  └────────────────────────────────────────────────┘     │
│                                                          │
│  ┌──────────────── twitch.tv のページ ──────────────┐    │
│  │                                                  │    │
│  │  [ISOLATED world]      [MAIN world]              │    │
│  │  content.js            worker-hook.js            │    │
│  │  DOM広告を除去          Worker をラップ            │    │
│  │                            │                     │    │
│  │                            ▼                     │    │
│  │                     ┌──────────────┐             │    │
│  │                     │ HLS Worker   │             │    │
│  │                     │ (注入済み)    │             │    │
│  │                     │ m3u8をフック  │             │    │
│  │                     └──────────────┘             │    │
│  └──────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

## 各コンポーネントの仕組み

### 1. worker-hook.js — 動画広告(SSAI)の差し替え

この拡張機能の中核です。**MAIN world**（Twitch のページ自身の JavaScript と同じ世界）で `document_start` に実行されます。

#### なぜ Worker をフックするのか

Twitch は動画（HLS）の再生処理を **Web Worker** の中で行っています。Web Worker はメインページとは別スレッドで動く独立した実行環境で、**メインページから `fetch` を上書きしても Worker 内の通信には影響しません**。そのため、広告の入った `.m3u8` プレイリストを取得しているのが Worker であり、そこに直接介入する必要があります。

#### ステップ①：Worker コンストラクタのラップ

```js
window.Worker = class extends OriginalWorker {
  constructor(url, options) {
    // 元ワーカーのソースを取得
    // → フックコードを先頭に前置
    // → blob 化して差し替え
    super(scriptURL, options);
  }
};
```

Twitch が新しい Worker を作ろうとした瞬間を捕まえ、元のワーカーコードの**先頭に広告除去コードを埋め込んで**から起動させます。これにより、以降の処理は「広告除去機能が組み込まれた Worker」として動きます。

#### ステップ②：Worker 内で `fetch` をフック

注入されたコードは、Worker 内の `self.fetch` を上書きします。監視対象は2種類の HLS プレイリストです。

- **master プレイリスト**（`usher.ttvnw.net/api/channel/hls/<channel>.m3u8`）
  → URL からチャンネル名を抽出し、含まれる各画質(variant)の URL を記録
- **variant プレイリスト**（実際の映像セグメント一覧）
  → 広告が含まれていないかを毎回チェック

#### ステップ③：広告の検出

variant プレイリストのテキスト内に、以下のような広告マーカーがあるかを調べます。

```
#EXT-X-DATERANGE:...,CLASS="twitch-stitched-ad",...
```

`stitched` / `Amazon` / `stitched-ad` といった文字列が含まれていれば、広告が挿入されていると判定します。

#### ステップ④：広告なしストリームへ差し替え

広告を検出したら、`gql.twitch.tv`（Twitch の GraphQL API）に対して **別のプレイヤータイプ**（`thunderdome`）で新しいアクセストークンを要求します。特定のプレイヤータイプでは Twitch が広告を挿入せずにストリームを返すことがあり、これを利用します。

```
広告あり variant を検出
    │
    ▼
GraphQL で新トークン取得 (playerType: "thunderdome")
    │
    ▼
usher から「広告なし」master → variant を取得
    │
    ▼
広告なしプレイリストを Worker に返す（本編だけ再生される）
```

差し替えに失敗した場合のフォールバックとして、広告を示す `#EXT-X-DATERANGE` 行を除去したプレイリストを返します。

> **補足**: この「別プレイヤータイプでトークンを取り直す」手法は、オープンソースの [TwitchAdSolutions](https://github.com/pixeltris/TwitchAdSolutions)（`video-swap-new` 方式）で知られる実績のあるアプローチをベースにしています。

### 2. rules.json — ネットワークブロック

`declarativeNetRequest` を使い、**ブラウザ層**で広告・トラッキング系ドメインへの通信を遮断します。JavaScript を実行する前にブラウザ自身がブロックするため軽量・確実です。

ブロック対象の例：

| ドメイン | 内容 |
|---|---|
| `amazon-adsystem.com` | Amazon（Twitch親会社）の広告システム |
| `imasdk.googleapis.com` | Google の動画広告SDK (IMA) |
| `doubleclick.net` / `googlesyndication.com` | Google 広告配信 |
| `adnxs.com` / `serving-sys.com` | 広告ネットワーク |
| `moatads.com` / `scorecardresearch.com` | 広告計測・解析 |

各ルールは `resourceTypes`（script / image / media など）で対象を絞っています。

---

### 3. content.js — DOM 上の広告除去

**ISOLATED world** で動作し、ページ内に表示される広告 UI を除去します。2段構えです。

1. **CSS 注入** — 広告要素のセレクタ（`[data-a-target="video-ad-label"]`、`.stream-display-ad` など）を `display: none` で非表示化
2. **MutationObserver** — Twitch は SPA（ページ遷移してもリロードしない）なので、DOM の変化を監視し、後から追加された広告オーバーレイもリアルタイムに除去

これにより「広告表示中」ラベルやページ内バナー広告枠が消えます。

---

### 4. popup — 状態表示

ツールバーアイコンをクリックすると表示される小さな UI です。

- 広告ブロックの各機能（動画広告 / 表示広告 / トラッキング）の状態を表示
- **「Twitch を再読み込み」ボタン** — 開いている Twitch タブをまとめてリロード（Twitch の仕様変更等で一時的に効かなくなった際の復旧用）

## 処理フロー図

配信ページを開いてから広告が差し替えられるまでの流れ：

```
ユーザーが twitch.tv/<channel> を開く
        │
        ▼
document_start で worker-hook.js が MAIN world に注入される
        │  → window.Worker をラップ
        ▼
Twitch のプレイヤーが HLS Worker を生成
        │  → ラップにより広告除去コードが注入された状態で起動
        ▼
Worker が usher から master .m3u8 を取得
        │  → チャンネル名と variant URL を記録
        ▼
Worker が variant .m3u8 を定期取得
        │
        ├─ 広告なし → そのまま再生
        │
        └─ 広告あり(stitched検出)
                │
                ▼
        GraphQL で別プレイヤータイプのトークン取得
                │
                ▼
        広告なしストリームを取得して差し替え
                │
                ▼
        本編のみが途切れず再生される
```

## インストール方法

1. Chrome で `chrome://extensions` を開く
2. 右上の **「デベロッパーモード」** を ON にする
3. **「パッケージ化されていない拡張機能を読み込む」** をクリック
4. この `extension` フォルダを選択
5. Twitch を開いて（既に開いていれば再読み込みして）動作を確認

## 権限の説明

`manifest.json` で要求している権限とその理由：

| 権限 | 用途 |
|---|---|
| `declarativeNetRequest` | 広告ドメインへの通信をブロックするため |
| `storage` | 設定値の保存（将来の拡張用） |
| `scripting` | スクリプト注入の補助 |
| `tabs` | ポップアップから Twitch タブを再読み込みするため |
| `host_permissions: *.twitch.tv, *.ttvnw.net` | Twitch ページとストリーム配信元に介入するため |

## 制限事項・既知の問題

- **将来効かなくなる可能性** — Twitch は広告回避対策を頻繁に更新します。特定プレイヤータイプでの広告なしストリーム取得が塞がれると、`worker-hook.js` のトークン取得ロジック調整が必要になります。
- **差し替え時の一瞬の乱れ** — 広告なしストリームへ切り替わる瞬間、数秒の黒画面や画質の切り替わりが発生することがあります。
- **Chrome Web Store 非対応** — 広告ブロッカーの公開制限があるため、これは個人利用の未署名拡張として「デベロッパーモード」で読み込む形になります。
- **同期 XHR の使用** — Worker ソース取得にコンストラクタ内で同期 XHR を使っています（この文脈では必要な設計上の妥協点です）。

## トラブルシューティング

**広告がまだ表示される**
1. ポップアップの「Twitch を再読み込み」を押す
2. `chrome://extensions` で拡張機能を一度リロードする
3. ブラウザの DevTools コンソールで `[TwitchAdblock]` のログを確認する

**映像が全く再生されない**
- 広告なしストリームの取得に失敗している可能性があります。一度拡張機能を無効化して切り分けてください。

**コンソールログの見方**
- `Worker constructor wrapped` … Worker のラップ成功
- `worker hook installed` … Worker 内フックの注入成功
- `広告を検出。広告なしストリームに切り替えます。` … 差し替え発動

## 用語集

| 用語 | 意味 |
|---|---|
| **SSAI** | Server-Side Ad Insertion。広告を配信サーバー側で本編ストリームに直接つなぎ込む方式 |
| **HLS** | HTTP Live Streaming。`.m3u8` プレイリストと映像セグメントで構成される配信方式 |
| **m3u8** | HLS のプレイリストファイル。再生する映像セグメントの一覧を記述 |
| **master プレイリスト** | 各画質(variant)へのリンクをまとめた親プレイリスト |
| **variant プレイリスト** | 特定画質の実際の映像セグメント一覧 |
| **usher** | Twitch の HLS プレイリスト配信サーバー（`usher.ttvnw.net`） |
| **GraphQL / gql** | Twitch の API エンドポイント（`gql.twitch.tv`）。アクセストークン取得に使用 |
| **MAIN world / ISOLATED world** | Chrome 拡張のスクリプト実行環境。MAIN はページ本体と同じ、ISOLATED は分離された安全な環境 |
| **declarativeNetRequest** | Chrome 拡張が宣言的にネットワークリクエストをブロック/改変する仕組み |
